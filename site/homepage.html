Browser Mirror

Browser Mirror is a way of sharing your browser session with anyone else. You share your session and you'll get a link which you can give to another person — they'll see what you see, and (with limits) they can interact with the page at the same time as you.

How Does It Work?

Browser Mirror is different than a typical screen-sharing applications. Instead of sharing the image that you see on the screen, it shares the page that you are looking at. It's also not the same as sharing the URL — the other person sees what you are looking at, they aren't just following along with the same URL. So you can look at a private page on an intranet, or something that is bound to a session, like airplane ticket purchases. As you continue to surf, the other person will follow along. If you are using a dynamic application, the other person will see the results as you interact with the application.

In most cases you can both interact with forms, though in some fancier applications your friend may not be able to use the full functionality of some parts of the page.

In technical terms, what Browser Mirror does is send the DOM to the other person. That is more or less the same as sending the HTML of the page — but without any of the special information the server uses to identify you, like cookies or Javascript state. Only the visible parts of the page are sent. This is somewhat like a screen sharing program, which sends pixels instead of DOM elements. Currently the elements are captured at regular intervals, and diffs are sent to cut down on traffic.

The Code

The code is hosted on github.

The code contains a Firefox Add-on, a simple Node.js server that handles communication between the two browsers, and a web site for the client to view.
